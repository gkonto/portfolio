Monkey in C++

A modern C++ implementation of the Monkey programming language from Writing an Interpreter in Go, built as a tree-walking interpreter with a Pratt parser, explicit AST, REPL, tests, and a lot of runtime-focused optimization work.

view source

stack: C++ / CMake / Catch2 / interpreter internals

Overview

This project follows the familiar Monkey language structure from Thorsten Ball’s book, but the implementation is written in modern C++ and pushes harder on runtime behavior than the original learning-oriented version.

The core flow stays clean: source text becomes tokens, the Pratt parser produces an AST, a resolver assigns indexed symbols, and the evaluator walks the tree to compute runtime values.

Language support

  • Integers, booleans, prefix and infix expressions.
  • if / else, let, and return.
  • Functions and closures.
  • Strings, arrays, hashes, indexing, and builtins like len, first, last, rest, and push.

Interpreter structure

  • A Pratt parser keeps precedence handling compact and extensible.
  • An explicit AST gives the evaluator a clear semantic structure to walk.
  • A resolver assigns indexed symbols for globals, locals, and builtins.
  • The project includes a REPL and a Catch2-based test suite.

Runtime work

  • Moved away from heavier runtime AST dispatch toward explicit node kinds.
  • Replaced string-based identifier lookup in the hot path with indexed symbol resolution.
  • Uses a tagged Value model so immediates like integers, booleans, and null do not always become heap-backed objects.
  • Splits temporary and persistent composite storage with a scratch arena plus persistent store.
  • Separates temporary call frames from escaping closure state.
  • Adds a small-arity argument fast path for common call shapes.

Performance notes

  • Local fibonacci(33) measurements are around ~1.3s - 1.6s with the optimized preset.
  • Peak memory is reported around ~1.6MB - 1.8MB.
  • The main remaining bottleneck is still the nature of recursive AST evaluation, not a single obvious memory-management issue.

Build, test, run

cmake --preset release-native-lto
cmake --build --preset release-native-lto
ctest --preset release-native-lto --output-on-failure
./build/release-native-lto/monkey_repl
./build/release-native-lto/monkey_performance_tests "Performance fibonacci(33)"
